그림자의 원리

Three.js의 그림자는 Shadow Map 기법을 사용합니다. 조명 시점에서 씬을 한 번 더 렌더링하여 깊이 정보를 텍스처에 저장하고, 이를 실제 렌더링 시 비교하여 그림자 영역을 결정합니다.

  그림자 렌더링 과정:

  1단계: 조명 시점에서 깊이 렌더링
         light ──→ [depth texture]

  2단계: 카메라 시점에서 실제 렌더링
         각 픽셀마다:
         "이 픽셀이 조명에서 보이는가?"
         → 보임: 밝음
         → 안 보임 (다른 물체에 가림): 그림자
그림자는 성능 비용이 큽니다. 조명마다 씬을 한 번 더 렌더링하기 때문입니다. 그림자가 필요한 조명과 오브젝트에만 선택적으로 활성화하세요.

그림자 활성화 3단계

그림자를 표시하려면 3곳에서 모두 활성화해야 합니다:

// 1. Renderer에서 그림자 활성화
renderer.shadowMap.enabled = true

// 2. 조명에서 그림자 생성 활성화
light.castShadow = true

// 3. 오브젝트에서:
mesh.castShadow = true     // 그림자를 드리움
mesh.receiveShadow = true  // 그림자를 받음 (바닥 등)

하나라도 빠지면 그림자가 보이지 않습니다. 가장 흔한 실수는 receiveShadow를 바닥에 설정하지 않는 것입니다.

Shadow Map 종류

캔버스 하단의 Shadow Type 버튼으로 4종을 비교할 수 있습니다:

  종류              품질    성능    특징
  ──────────────────────────────────────────
  BasicShadowMap    낮음    빠름    계단 현상 심함
  PCFShadowMap      중간    보통    기본 안티앨리어싱
  PCFSoftShadowMap  좋음    보통    부드러운 경계 (기본 추천)
  VSMShadowMap      좋음    느림    블러 효과, 빛 새는 현상 가능
PCFSoftShadowMap이 품질/성능 밸런스가 가장 좋아 기본 추천입니다. shadow.radius로 블러 정도를 조절할 수 있습니다 (PCFSoft 전용).

Shadow Map 해상도

shadow.mapSize는 깊이 텍스처의 해상도입니다. mapSize 버튼으로 256~2048을 비교해보세요:

light.shadow.mapSize.width = 1024
light.shadow.mapSize.height = 1024
  mapSize    그림자 품질     GPU 메모리
  ────────────────────────────────────
  256        계단 심함       적음
  512        보통            보통
  1024       좋음 (추천)     보통
  2048       매우 좋음       많음
  4096       최고            매우 많음

값은 항상 2의 거듭제곱(256, 512, 1024...)이어야 합니다.

Shadow Camera (그림자 카메라)

DirectionalLight의 그림자는 내부적으로 OrthographicCamera를 사용합니다. 이 카메라의 frustum 범위가 그림자가 렌더링되는 영역을 결정합니다:

dirLight.shadow.camera.left = -5
dirLight.shadow.camera.right = 5
dirLight.shadow.camera.top = 5
dirLight.shadow.camera.bottom = -5
dirLight.shadow.camera.near = 0.5
dirLight.shadow.camera.far = 20

Shadow Camera 버튼으로 이 frustum을 시각화할 수 있습니다.

shadow camera의 범위가 너무 넓으면 해상도가 낭비되고 그림자가 흐려집니다. 너무 좁으면 범위 밖의 그림자가 잘립니다. 씬에 맞게 가능한 한 타이트하게 설정하는 것이 품질의 핵심입니다.

그림자를 지원하는 조명

모든 조명이 그림자를 생성할 수 있는 것은 아닙니다:

  조명               그림자   Shadow Camera 타입
  ──────────────────────────────────────────────
  DirectionalLight   ✓       OrthographicCamera
  SpotLight          ✓       PerspectiveCamera
  PointLight         ✓       CubeCamera (6방향)
  AmbientLight       ✗       방향이 없음
  HemisphereLight    ✗       방향이 없음

PointLight는 6방향 큐브맵을 렌더링하므로 6배의 비용이 듭니다. 가능하면 SpotLight로 대체하세요.

그림자 최적화 팁

  • 그림자를 생성하는 조명 수를 최소화 (보통 1~2개)
  • castShadow는 필요한 오브젝트만 활성화
  • shadow camera frustum을 씬에 타이트하게 맞추기
  • mapSize는 1024이면 대부분 충분
  • 움직이지 않는 씬은 그림자를 텍스처에 베이크 (bake) 고려

전체 코드

1단계: Renderer에서 그림자 활성화

const renderer = new THREE.WebGLRenderer({ canvas, antialias: true })
renderer.shadowMap.enabled = true
renderer.shadowMap.type = THREE.PCFSoftShadowMap
// PCFSoftShadowMap: 부드러운 그림자 경계 (추천)

이 설정 없이는 어떤 그림자도 표시되지 않습니다.

2단계: DirectionalLight 그림자

const dirLight = new THREE.DirectionalLight(0xffffff, 2)
dirLight.position.set(-2, 6, 3)
dirLight.castShadow = true  // 그림자 생성 활성화

// 그림자 품질 설정
dirLight.shadow.mapSize.width = 1024   // 해상도
dirLight.shadow.mapSize.height = 1024
dirLight.shadow.radius = 4  // 블러 반경 (PCFSoft 전용)

// 그림자 카메라 범위 (OrthographicCamera)
dirLight.shadow.camera.left = -5
dirLight.shadow.camera.right = 5
dirLight.shadow.camera.top = 5
dirLight.shadow.camera.bottom = -5
dirLight.shadow.camera.near = 0.5
dirLight.shadow.camera.far = 20

scene.add(dirLight)

shadow.camera의 범위를 씬에 맞게 조절하는 것이 품질의 핵심입니다. 범위가 넓으면 같은 mapSize에서 해상도가 분산되어 품질이 떨어집니다.

3단계: 오브젝트에 그림자 설정

// 그림자를 드리우는 오브젝트
box.castShadow = true
sphere.castShadow = true

// 그림자를 받는 오브젝트 (바닥)
floor.receiveShadow = true

// 둘 다 가능 (다른 물체의 그림자를 받으면서 자신도 드리움)
box.castShadow = true
box.receiveShadow = true

castShadow를 활성화한 오브젝트만 그림자를 드리우므로, 성능을 위해 필요한 오브젝트에만 설정하세요.

4단계: SpotLight 그림자

const spotLight = new THREE.SpotLight(
  0xff8844, 5, 15,
  Math.PI / 5,  // 원뿔 각도
  0.4           // penumbra
)
spotLight.position.set(4, 5, 2)
spotLight.castShadow = true

// SpotLight의 shadow camera는 PerspectiveCamera
spotLight.shadow.mapSize.width = 1024
spotLight.shadow.mapSize.height = 1024
spotLight.shadow.camera.near = 1
spotLight.shadow.camera.far = 15

scene.add(spotLight)
scene.add(spotLight.target)  // target 씬에 추가 필수

5단계: Shadow Camera 디버깅

// DirectionalLight의 shadow camera를 시각화
const helper = new THREE.CameraHelper(
  dirLight.shadow.camera
)
scene.add(helper)
// 프로덕션에서는 제거

그림자가 잘리거나 품질이 나쁠 때 shadow camera의 범위를 시각화하여 확인합니다. frustum 밖의 영역에는 그림자가 생기지 않습니다.

6단계: Shadow Map 타입 변경 (런타임)

renderer.shadowMap.type = THREE.PCFSoftShadowMap

// 런타임에 변경 시 material 재컴파일 필요
scene.traverse((obj) => {
  if (obj instanceof THREE.Mesh) {
    obj.material.needsUpdate = true
  }
})

// shadow map도 재생성
light.shadow.map?.dispose()
light.shadow.map = null

shadow map 타입이나 mapSize를 런타임에 바꾸면 기존 shadow map을 dispose하고 material을 갱신해야 합니다.
