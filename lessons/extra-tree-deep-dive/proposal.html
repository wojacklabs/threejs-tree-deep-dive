줄기 절차적 교체 제안

GLB 줄기 → 절차적 줄기 교체 제안

현재 GLB 줄기는 단순 원기둥에 불과합니다. 절차적으로 교체하면 더 풍부한 형태와 질감을 구현할 수 있습니다. 5단계 Phase로 캔버스에서 비교하며 확인하세요.

Phase 1: 기본 교체

GLB trunk를 CylinderGeometry로 교체. 외관 차이 없는 수준.

  • CylinderGeometry(radial × height segments)
  • bend/twist 기본 적용
  • wind alpha 인코딩 동일
  • bush(수관) merge는 그대로 유지

Phase 2: 뿌리 노출 (Root Flare)

바닥 근처 vertex를 바깥+아래로 확장하여 자연스러운 뿌리 형태.

  • 바닥 25% 영역에만 적용
  • 비대칭 노이즈로 뿌리 방향 변화
  • rootFlare 파라미터로 강도 조절
실제 나무는 바닥에서 갑자기 직선으로 시작하지 않습니다. 뿌리가 벌어지는 형태가 안정감과 사실감을 줍니다.

Phase 3: 표면 질감

vertex displacement로 나무껍질 디테일 추가.

  • 세로 홈 (groove): sin(angle × freq) — 나무껍질 갈라진 골
  • 옹이 (knot): 특정 높이에 반지름 돌출 — 가지 흔적
  • 위로 갈수록 약해짐 (윗부분은 매끄러움)

Phase 4: 분기 줄기

단일 원기둥이 아닌 Y자 분기. 나무의 구조적 다양성.

  • 분기점 높이/각도 seed 기반
  • 아래: 공통 줄기 (두껍게)
  • 위: 2개 분기 (각각 다른 두께/길이)
  • 각 분기 끝에 bush(수관) 연결 가능

Phase 5: 셰이더 색상

단일 uniform 색상 대신 vertex color + 셰이더로 풍부한 표현.

  • 높이별 그라데이션: 바닥 어둡고, 위로 밝아짐
  • 이끼/지의류: 바닥 녹색 패치 (노이즈 기반)
  • 세로 줄무늬: 나무껍질 결 표현
  • wrap lighting + hemisphere 셰이딩

구현 순서

  Phase 1 → 2 → 3 → 4 → 5

  각 Phase는 이전 Phase를 포함합니다:
  Phase 3 = Phase 1 + 2 + 3

  캔버스에서 5개를 나란히 비교하면서
  lil-gui로 파라미터를 조절해보세요.

성능 비교

  현재 GLB trunk:  ~9,366 vertices (과도함)
  절차적 16×24:    ~425 vertices (충분)
  분기 포함:       ~1,200 vertices (여유)

  100그루 기준:
  GLB:    936,600 vertices
  절차적: 42,500~120,000 vertices
  → 7~22배 절약

Phase별 핵심 코드

Phase 2: 뿌리 노출

// 바닥 25%에만 적용
const rootT = max(0, 1 - t * 4)
const flare = 1 + rootT² × rootFlare × 10

// 비대칭 노이즈
const rootAngle = atan2(z, x)
const rootNoise = sin(rootAngle × 3 + seed)
x *= flare × (1 + rootT × rootNoise × rootFlare)
z *= flare

Phase 3: 세로 홈

const angle = atan2(z, x)
const groove = sin(angle × grooveFreq + seed) × grooveDepth
const r = sqrt(x² + z²)
const newR = r + groove × (1 - t × 0.5)
x = x/r × newR
z = z/r × newR

Phase 4: Y자 분기

// 분기점 아래: 공통 줄기
buildCylinder(thickness×1.3, thickness, splitH)

// 분기점 위: 2개 분기
buildCylinder(thickness×0.9, ..., startY=splitH,
  offsetX = -sin(splitAngle) × ...)
buildCylinder(thickness×0.8, ..., startY=splitH,
  offsetX = +sin(splitAngle) × ...)

Phase 5: 이끼 + 줄무늬

// 이끼 (바닥에 녹색)
const mossT = max(0, 1 - t×2.5) × mossNoise × mossAmount
color = mix(barkColor, mossColor, mossT)

// 세로 줄무늬
const stripe = sin(angle×12 + noise(x,y,z)×2)
color -= stripe × 0.08 × colorVariation
