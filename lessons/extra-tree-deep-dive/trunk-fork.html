줄기 분기 (Fork) 탐색

현재 문제

  현재 구조:
  │         ← 줄기 꼭대기 (어색함)
  │╲
  │ ╲ 가지
  │  ╲
  │
  │╱
  │   가지
  │
  ┃ 줄기
  ┃

  줄기가 맨 꼭대기까지 일직선으로 올라감
  → 분기점 이후에도 줄기가 남아있어 부자연스러움

개선 방향: 분기 (Fork)

     ╱╲         ← 분기 후 두 가지로 갈라짐
    ╱  ╲
   ╱    ╲
  ├──────┤     ← 분기점 (forkHeight)
  │      │
  │      │
  ┃      ┃     ← 줄기 (분기점까지만)
  ┃      ┃

  분기점에서 줄기 종료,
  2~3개 주가지로 갈라짐

분기 파라미터

  • 분기 높이: 줄기의 몇 %에서 갈라지는지
  • 분기 수: 2개 (Y자) 또는 3개
  • 벌어짐 각도: 분기된 가지가 얼마나 벌어지는지
  • 분기 가지 길이: 분기 후 가지의 길이
  • 분기 가지 두께: 줄기 대비 비율
  • 비대칭: 한쪽이 더 크거나 긴 변이

줄기-분기 연결

분기점에서 줄기 두께가 자연스럽게 두 가지로 나뉘어야 합니다. 줄기 단면적 ≈ 분기 가지들의 단면적 합 (파이프 모델).

조절 가이드

  • 분기 높이 낮게: 키 작고 넓게 퍼진 나무 (참나무)
  • 분기 높이 높게: 키 크고 좁은 나무 (소나무)
  • 벌어짐 크게: 수관이 넓은 나무
  • 비대칭 높게: 자연스러운 불규칙함

분기 구현 핵심

파이프 모델 (두께 보존)

// 줄기 단면적 = 분기 가지들의 단면적 합
// A_trunk = A_branch1 + A_branch2
// π r² = π r1² + π r2²
// r_branch = r_trunk / sqrt(N)
const branchRadius = trunkRadius / Math.sqrt(forkCount)

분기 방향

// 분기점에서 줄기 접선 기준으로 벌어짐
const forkDir = trunkTangent.clone()
  .applyAxisAngle(spreadAxis, spreadAngle)
  .applyAxisAngle(trunkTangent, azimuth)
