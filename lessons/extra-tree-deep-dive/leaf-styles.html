잎 스타일 비교

4가지 잎 스타일 비교

같은 줄기+가지 뼈대에 다른 잎 방식을 적용하여 비교합니다. 🎲 버튼으로 다른 seed를 시도해보세요.

A. 리얼리스틱 (개별 잎 카드)

  • PlaneGeometry에 잎 모양 텍스처 (Canvas 생성)
  • 가지 끝에 수백 장 배치 (구면 분포)
  • alphaTest로 잎 모양 cutout
  • 개별 잎의 방향/크기 랜덤
  장점: 가장 사실적, 개별 잎 보임
  단점: draw call 폭발 (잎 수백 개 × 나무 수)
  성능: ⭐ (많은 나무에 비실용적)

B. 스타일라이즈드 (구/타원 클러스터)

  • IcosahedronGeometry를 납작하게 + 노이즈 변형
  • flatShading으로 만화풍
  • 가지 끝에 2~4개 덩어리
  • 색상 변형으로 다양성
  장점: 성능 최고, 스타일 일관성
  단점: 잎 느낌 약함, 실루엣 단순
  성능: ⭐⭐⭐⭐⭐

C. 빌보드 클러스터 (다중 잎 텍스처)

  • 여러 잎이 그려진 텍스처 1장을 PlaneGeometry에 적용
  • 카드 수십 장을 겹쳐서 수관 형성
  • 카드당 여러 잎이 보여서 효율적
  • 대부분의 게임이 사용하는 방식
  장점: 성능/퀄리티 균형, 업계 표준
  단점: 가까이 보면 평면 카드 보임
  성능: ⭐⭐⭐⭐

D. 셰이더 기반 (구체 + cutout)

  • SphereGeometry에 셰이더로 잎 패턴 생성
  • 노이즈 기반 discard로 잎 형태
  • 프레넬 림, 색상 변화 셰이더 내장
  • geometry 최소, 셰이더가 모든 것 처리
  장점: geometry 비용 최소, 유연한 표현
  단점: 실루엣이 구체 형태, 가까이 보면 패턴
  성능: ⭐⭐⭐⭐

각 스타일 핵심 코드

A. 개별 잎 카드

// Canvas로 잎 텍스처 생성
ctx.beginPath()
ctx.bezierCurveTo(...)  // 잎 윤곽
ctx.fill()
// + 잎맥 stroke

// PlaneGeometry에 적용
const leafGeo = new PlaneGeometry(0.15, 0.18)
material.map = leafTexture
material.alphaTest = 0.3
// 가지 끝에 30~50장 구면 배치

B. 스타일라이즈드

const geo = new IcosahedronGeometry(radius, 2)
// Y축 눌러서 납작하게
pos.setY(i, y * 0.7)
// 노이즈로 울퉁불퉁
pos *= 1 + sin(x*10) * cos(z*10) * 0.15
material.flatShading = true

C. 빌보드 클러스터

// 여러 잎이 그려진 텍스처 1장
for (12 leaves) {
  ctx.ellipse(x, y, w, h, rot, 0, 2π)
}
// PlaneGeometry 카드 6~10장 겹침
// 각 카드 랜덤 방향 회전

D. 셰이더

// fragment shader에서 잎 패턴
float pattern = noise(worldPos * 8.0)
             + noise(worldPos * 16.0) * 0.3;
if (pattern < 0.5) discard;

// 색상 변화
color = mix(green1, green2, pattern);
// + fresnel rim + 조명
