Raycasting이란?

3D 공간에서 보이지 않는 광선(ray)을 쏘아 어떤 오브젝트와 교차하는지 검사하는 기법입니다.

  마우스 클릭
      │
      ▼
   카메라 ─────────── ray ──────────→
              │             │
              │   ┌─────┐   │
              │   │ hit │   │
              │   └─────┘   │
              │             │
              └─────────────┘
  "이 광선이 어떤 오브젝트와 만나는가?"

캔버스에서 오브젝트 위에 마우스를 올려보세요. Ray 버튼으로 광선을 시각화할 수 있습니다.

핵심 과정 3단계

1. 마우스 좌표를 정규화 (-1 ~ +1)

const mouse = new THREE.Vector2()

function updateMouse(event: MouseEvent) {
  const rect = canvas.getBoundingClientRect()
  // 화면 좌표 → NDC (Normalized Device Coordinates)
  mouse.x = ((event.clientX - rect.left) / rect.width) * 2 - 1
  mouse.y = -((event.clientY - rect.top) / rect.height) * 2 + 1
  //  ↑ y축은 반전 (화면은 아래가 +, NDC는 위가 +)
}
  화면 좌표:          NDC:
  (0,0)──→ x         (-1,+1)──→(+1,+1)
  │                      │
  ▼ y                    ▼
  (w,h)              (-1,-1)──→(+1,-1)

2. 카메라에서 광선 생성

const raycaster = new THREE.Raycaster()
raycaster.setFromCamera(mouse, camera)
// mouse(NDC)와 camera로부터 광선의 origin과 direction을 계산

3. 교차 검사

const intersects = raycaster.intersectObjects(meshes)

if (intersects.length > 0) {
  const hit = intersects[0]  // 가장 가까운 교차
  hit.object    // 교차한 오브젝트
  hit.point     // 교차 지점 (Vector3)
  hit.distance  // 카메라에서의 거리
  hit.face      // 교차한 면
  hit.faceIndex // 면 인덱스
}
intersectObjects는 거리순으로 정렬된 배열을 반환합니다. intersects[0]이 가장 가까운(눈에 보이는) 오브젝트입니다.

호버 효과

캔버스에서 오브젝트 위에 마우스를 올리면 색이 밝아집니다:

canvas.addEventListener('mousemove', (e) => {
  updateMouse(e)

  const intersects = raycaster.intersectObjects(meshes)

  if (intersects.length > 0) {
    // 호버 중 — 색 밝게
    material.color.multiplyScalar(1.4)
    canvas.style.cursor = 'pointer'
  } else {
    // 호버 아님 — 원래 색
    material.color.copy(originalColor)
    canvas.style.cursor = 'default'
  }
})

왼쪽 상단 info bar에서 호버 중인 오브젝트의 이름, 거리, 교차 지점, face 인덱스를 확인하세요.

클릭 선택

클릭하면 오브젝트가 선택되어 emissive(발광) 효과가 적용되고 위아래로 바운스합니다. 같은 오브젝트를 다시 클릭하면 선택 해제됩니다.

canvas.addEventListener('click', () => {
  if (hoveredObject) {
    // emissive로 "선택됨" 표시
    material.emissive.copy(originalColor)
    material.emissiveIntensity = 0.4
  }
})

intersectObjects 옵션

// 기본: 직접 전달한 오브젝트만 검사
raycaster.intersectObjects(meshes)

// recursive: true → 자식까지 검사 (Group, GLTF 모델에 필수)
raycaster.intersectObjects([group], true)

// 단일 오브젝트
raycaster.intersectObject(mesh)
GLTF 모델은 보통 Group 안에 여러 Mesh가 들어있으므로 recursive: true를 사용해야 합니다. Lesson 12에서 다룹니다.

Raycasting 성능

오브젝트 수가 많으면 교차 검사가 느려질 수 있습니다:

  오브젝트 수    방법
  ──────────────────────────────────
  ~100개        intersectObjects 그대로 사용
  ~1000개       검사 대상을 줄임 (가까운 것만)
  ~10000개      BVH (Bounding Volume Hierarchy) 사용
               → three-mesh-bvh 라이브러리

내부적으로 Three.js는 각 오브젝트의 bounding sphere를 먼저 검사하여 명백히 교차하지 않는 것을 빠르게 걸러냅니다.

Raycasting 활용

  • 클릭 선택 — 오브젝트 클릭, UI 인터랙션
  • 호버 효과 — 마우스 오버 시 하이라이트, 툴팁
  • 바닥 클릭 — 캐릭터 이동, 오브젝트 배치
  • 충돌 감지 — 간단한 시선 검사, 장애물 감지
  • 높이 맞추기 — 지형 위에 오브젝트 배치 (아래로 ray)

전체 코드

1단계: Raycaster 생성

const raycaster = new THREE.Raycaster()
const mouse = new THREE.Vector2()

// 마우스 좌표를 NDC로 변환
function updateMouse(event: MouseEvent) {
  const rect = canvas.getBoundingClientRect()
  mouse.x = ((event.clientX - rect.left) / rect.width) * 2 - 1
  mouse.y = -((event.clientY - rect.top) / rect.height) * 2 + 1
}

getBoundingClientRect()를 사용하면 캔버스가 페이지의 어디에 있든 정확한 좌표를 계산합니다.

2단계: mousemove — 호버 감지

let hoveredObject: InteractiveObject | null = null

canvas.addEventListener('mousemove', (e) => {
  updateMouse(e)

  // 카메라에서 마우스 방향으로 광선 생성
  raycaster.setFromCamera(mouse, camera)

  // 인터랙티브 오브젝트들과 교차 검사
  const meshes = interactives.map(i => i.mesh)
  const intersects = raycaster.intersectObjects(meshes)

  // 이전 호버 해제
  if (hoveredObject) {
    material.color.copy(hoveredObject.originalColor)
    canvas.style.cursor = 'default'
    hoveredObject = null
  }

  // 새 호버
  if (intersects.length > 0) {
    const hit = intersects[0]
    const obj = interactives.find(i => i.mesh === hit.object)
    if (obj) {
      hoveredObject = obj
      material.color.multiplyScalar(1.4)  // 밝게
      canvas.style.cursor = 'pointer'
    }
  }
})

3단계: click — 선택/해제

let selectedObject: InteractiveObject | null = null

canvas.addEventListener('click', () => {
  if (hoveredObject) {
    if (selectedObject === hoveredObject) {
      // 같은 것 클릭 → 선택 해제
      material.emissive.setHex(0x000000)
      selectedObject = null
    } else {
      // 새 선택
      if (selectedObject) {
        // 이전 선택 해제
        prevMaterial.emissive.setHex(0x000000)
      }
      selectedObject = hoveredObject
      material.emissive.copy(originalColor)
      material.emissiveIntensity = 0.4
    }
  } else {
    // 빈 곳 클릭 → 전체 해제
    if (selectedObject) {
      material.emissive.setHex(0x000000)
      selectedObject = null
    }
  }
})

emissive는 조명과 무관하게 자체적으로 빛나는 색입니다. 선택 표시에 color보다 자연스럽습니다.

4단계: 교차점 시각화

// 교차 지점에 작은 구체 표시
const hitMarker = new THREE.Mesh(
  new THREE.SphereGeometry(0.05, 8, 8),
  new THREE.MeshBasicMaterial({ color: 0xff0000 })
)
scene.add(hitMarker)

// raycast 결과에서:
if (intersects.length > 0) {
  hitMarker.position.copy(intersects[0].point)
  hitMarker.visible = true
} else {
  hitMarker.visible = false
}

5단계: 레이 시각화

// 광선을 Line으로 시각화 (디버깅용)
const origin = raycaster.ray.origin.clone()
const end = origin.clone().add(
  raycaster.ray.direction.clone().multiplyScalar(20)
)
const geo = new THREE.BufferGeometry()
  .setFromPoints([origin, end])
const rayLine = new THREE.Line(geo,
  new THREE.LineBasicMaterial({ color: 0xff0000 })
)
