물리 엔진이란?

Three.js는 렌더링만 합니다. 중력, 충돌, 반발은 별도의 물리 엔진이 처리합니다.

  물리 엔진 (cannon-es)       Three.js
  ┌─────────────────┐        ┌─────────────────┐
  │ Body (물리체)     │  동기화  │ Mesh (시각체)    │
  │ position, quat  │ ──────→│ position, quat  │
  │ velocity, force │        │ geometry, mat   │
  │ shape, mass     │        │                 │
  └─────────────────┘        └─────────────────┘
       물리 계산                   렌더링

매 프레임: 물리 엔진이 위치 계산 → Three.js mesh에 복사 → 렌더링

cannon-es

JavaScript 물리 엔진입니다. cannon.js의 ES module 포크입니다:

npm install cannon-es
import * as CANNON from 'cannon-es'

대안으로 Rapier(WASM 기반, 더 빠름)가 있지만, cannon-es가 학습에 더 직관적입니다.

핵심 과정 3단계

1. 물리 월드 생성

const world = new CANNON.World()
world.gravity.set(0, -9.82, 0)  // 중력

2. Body 생성 (물리체)

const body = new CANNON.Body({
  mass: 1,        // 0이면 정적(바닥 등)
  shape: new CANNON.Sphere(0.5),
  position: new CANNON.Vec3(0, 5, 0),
})
world.addBody(body)

3. 매 프레임 동기화

// 물리 시뮬 스텝
world.step(1/60, delta, 3)

// body → mesh 동기화
mesh.position.copy(body.position)
mesh.quaternion.copy(body.quaternion)

Body 종류

  타입              mass    움직임    용도
  ──────────────────────────────────────────
  DYNAMIC (동적)    > 0     중력+충돌  공, 상자, 캐릭터
  STATIC (정적)     0       고정      바닥, 벽, 지형
  KINEMATIC         0       코드제어   움직이는 플랫폼, 엘리베이터

Shape (충돌 형태)

new CANNON.Sphere(radius)           // 구
new CANNON.Box(halfExtents)         // 직육면체
new CANNON.Plane()                  // 무한 평면
new CANNON.Cylinder(rTop, rBot, h)  // 원기둥
new CANNON.ConvexPolyhedron(...)    // 볼록 다면체 (복잡한 형태)

Shape은 렌더링 geometry와 별도입니다. 시각적으로 복잡한 모델도 물리에서는 단순한 shape(구, 박스)으로 근사합니다.

복잡한 GLTF 모델에 정확한 충돌체가 필요하면 ConvexPolyhedron 또는 Trimesh를 사용하지만, 성능 비용이 큽니다. 가능하면 기본 shape 조합으로 근사하세요.

재질 (Material) — 마찰과 반발

const material = new CANNON.Material('default')
const contact = new CANNON.ContactMaterial(
  material, material, {
    friction: 0.3,       // 마찰 (0=빙판, 1=고무)
    restitution: 0.4,    // 반발 (0=찰흙, 1=슈퍼볼)
  }
)
world.addContactMaterial(contact)

오른쪽 lil-gui 패널에서 restitution과 friction을 실시간으로 바꿔보세요. 공이 튀는 정도가 달라집니다.

힘과 속도

// 속도 직접 설정 (즉시)
body.velocity.set(10, 5, 0)

// 힘 적용 (가속)
body.applyForce(new CANNON.Vec3(100, 0, 0))

// 충격 (순간적 힘)
body.applyImpulse(new CANNON.Vec3(0, 10, 0))

Shoot! 버튼이 카메라 방향으로 구체를 발사합니다. shootForce 슬라이더로 힘을 조절해보세요.

고정 타임스텝

world.step(
  1/60,    // 고정 타임스텝 (60Hz)
  delta,   // 실제 경과 시간
  3        // 최대 서브스텝
)

물리 시뮬레이션은 고정 타임스텝으로 실행해야 안정적입니다. delta가 크면(FPS 떨어지면) 여러 번 나눠서 계산합니다.

lil-gui — Debug UI

이 레슨에서 처음 도입하는 디버그 패널입니다. 물리 파라미터를 실시간으로 조절하며 결과를 확인할 수 있습니다:

import GUI from 'lil-gui'

const gui = new GUI()

// 슬라이더
gui.add(params, 'gravity', -20, 0, 0.1)
   .onChange((v) => world.gravity.set(0, v, 0))

// 버튼
gui.add(params, 'addSphere').name('Drop Sphere')

// 폴더로 그룹핑
const folder = gui.addFolder('Physics')
folder.add(params, 'restitution', 0, 1)
lil-gui는 Three.js 개발의 필수 도구입니다. Material 색상, 조명 강도, 카메라 위치 등 어떤 값이든 슬라이더/컬러피커/체크박스로 실시간 조절할 수 있습니다. 코드를 수정하고 새로고침하는 것보다 훨씬 빠릅니다. 프로덕션에서는 제거하고, 개발 중에만 사용합니다.

캔버스 데모

  • 벽돌 탑 — 6층 × 3열, 쌓아둔 상자
  • 경사면 — 왼쪽의 기울어진 판
  • Drop Sphere/Box — 위에서 떨어뜨리기
  • Shoot! — 카메라 방향으로 구체 발사 (탑을 무너뜨려보세요)
  • Physics 폴더 — gravity, restitution, friction 실시간 조절
  • Reset Tower — 탑 재건

게임 캐릭터 이동 패턴

실제 게임에서는 물리 마찰로 캐릭터를 제어하지 않습니다. velocity를 직접 설정하고, 물리 엔진은 충돌과 중력만 담당합니다:

const characterBody = new CANNON.Body({
  mass: 70,
  shape: new CANNON.Sphere(0.5),
  fixedRotation: true,   // 넘어지지 않게 회전 잠금
  linearDamping: 0.95,   // 자체 감속 (마찰 대신)
})

// 캐릭터 재질: 마찰 0 (벽에 달라붙지 않게)
const charContact = new CANNON.ContactMaterial(
  charMaterial, groundMaterial,
  { friction: 0.0, restitution: 0.0 }
)

// 매 프레임: velocity로 직접 이동
if (wKey) body.velocity.z = -speed
else body.velocity.z = 0
// 중력/충돌은 물리 엔진이 자동 처리
물리 엔진은 환경(충돌+중력)만 담당, 이동은 게임 로직이 담당하는 것이 핵심 분리 원칙입니다. 물리에 전부 맡기면 빙판에서 미끄러지고, 경사면에서 흘러내리고, 벽에 달라붙는 문제가 생깁니다. fixedRotation: true는 캐릭터가 충돌 시 넘어지지 않게 하는 필수 설정입니다.

cannon-es 성능과 대안

cannon-es는 순수 JavaScript이므로 대규모에서 느립니다:

  body 수     cannon-es     Rapier (WASM)
  ──────────────────────────────────────
  ~50         60fps         60fps
  ~200        50fps         60fps
  ~500        30fps         60fps
  ~1000+      10fps         50fps+

프로덕션에서는 Rapier(@dimforge/rapier3d-compat)를 추천합니다. Rust로 작성되어 WASM으로 실행되며 3~5배 빠릅니다. API 패턴은 유사하므로 cannon-es의 개념을 이해하면 전환이 어렵지 않습니다.

  용도               추천
  ────────────────────────────
  학습/프로토타입     cannon-es (직관적, 자료 많음)
  프로덕션 게임       Rapier (빠름, CCD 지원)

전체 코드

1단계: 물리 월드 생성

import * as CANNON from 'cannon-es'

const world = new CANNON.World()
world.gravity.set(0, -9.82, 0)

// 충돌 감지 최적화
world.broadphase = new CANNON.SAPBroadphase(world)

// 재질 (모든 물체 공통)
const material = new CANNON.Material('default')
const contact = new CANNON.ContactMaterial(
  material, material,
  { friction: 0.3, restitution: 0.4 }
)
world.addContactMaterial(contact)
world.defaultContactMaterial = contact

SAPBroadphase는 정렬 기반 충돌 감지로 기본보다 빠릅니다.

2단계: 바닥 (정적 Body)

// Three.js 바닥
const floorMesh = new THREE.Mesh(
  new THREE.PlaneGeometry(20, 20),
  new THREE.MeshStandardMaterial({ color: 0x333333 })
)
floorMesh.rotation.x = -Math.PI / 2
scene.add(floorMesh)

// Cannon.js 바닥 (정적)
const floorBody = new CANNON.Body({
  type: CANNON.Body.STATIC,  // mass: 0과 동일
  shape: new CANNON.Plane(),
})
floorBody.quaternion.setFromEuler(-Math.PI / 2, 0, 0)
world.addBody(floorBody)

정적 Body는 mass: 0 또는 type: STATIC으로 설정합니다. 중력의 영향을 받지 않고 고정됩니다.

3단계: 동적 오브젝트 생성

function addSphere(x: number, y: number, z: number) {
  // Three.js 시각체
  const mesh = new THREE.Mesh(sphereGeo, material)
  mesh.castShadow = true
  scene.add(mesh)

  // Cannon.js 물리체
  const body = new CANNON.Body({
    mass: 1,
    shape: new CANNON.Sphere(0.4),
    position: new CANNON.Vec3(x, y, z),
  })
  world.addBody(body)

  // 쌍으로 저장
  objects.push({ mesh, body })
}

시각체(mesh)와 물리체(body)를 쌍으로 관리하는 것이 핵심 패턴입니다.

4단계: 매 프레임 동기화

function animate() {
  requestAnimationFrame(animate)

  // 물리 시뮬레이션
  world.step(1/60, delta, 3)

  // body → mesh 동기화
  for (const obj of objects) {
    obj.mesh.position.copy(obj.body.position)
    obj.mesh.quaternion.copy(obj.body.quaternion)
  }

  renderer.render(scene, camera)
}

position과 quaternion 두 가지를 복사해야 위치와 회전이 모두 반영됩니다.

5단계: 구체 발사

function shootSphere() {
  // 카메라 방향 벡터
  const dir = new THREE.Vector3()
  camera.getWorldDirection(dir)

  // 카메라 앞에서 생성
  const pos = camera.position.clone()
    .add(dir.clone().multiplyScalar(2))
  addSphere(pos.x, pos.y, pos.z)

  // 속도 설정 (방향 × 힘)
  const body = objects[objects.length - 1].body
  body.velocity.set(
    dir.x * force,
    dir.y * force,
    dir.z * force
  )
}

6단계: lil-gui 설정

import GUI from 'lil-gui'

const gui = new GUI()

// 숫자 슬라이더 (min, max, step)
gui.add(params, 'gravity', -20, 0, 0.1)
  .onChange((v) => world.gravity.set(0, v, 0))

// 버튼 (함수 호출)
gui.add(params, 'shootSphere').name('Shoot!')

// 폴더로 그룹핑
const folder = gui.addFolder('Physics')
folder.add(params, 'restitution', 0, 1, 0.05)

// 색상 (addColor)
gui.addColor(params, 'color')
  .onChange((v) => material.color.set(v))

// 체크박스 (boolean)
gui.add(params, 'wireframe')
  .onChange((v) => material.wireframe = v)

lil-gui는 값의 타입을 자동 감지하여 적절한 컨트롤을 생성합니다: number → 슬라이더, boolean → 체크박스, string → 입력, function → 버튼, color → 컬러피커.
