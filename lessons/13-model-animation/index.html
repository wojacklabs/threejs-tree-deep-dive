Three.js 애니메이션 시스템

GLTF 모델에 포함된 애니메이션(걷기, 뛰기, 대기 등)을 재생하는 시스템입니다. 3가지 핵심 클래스로 구성됩니다:

  AnimationClip     → 애니메이션 데이터 (키프레임)
       │
  AnimationAction   → 재생 제어 (play, pause, 속도, 무게)
       │
  AnimationMixer    → 여러 Action을 관리, delta로 업데이트

AnimationMixer

특정 오브젝트에 대한 애니메이션 재생기입니다. 모든 애니메이션 Action을 관리하고, 매 프레임 업데이트합니다:

// 모델에 대한 mixer 생성
const mixer = new THREE.AnimationMixer(model)

// 매 프레임 업데이트 (animate 내부)
mixer.update(delta)  // delta = clock.getDelta()
mixer.update(delta)를 호출하지 않으면 애니메이션이 재생되지 않습니다. Lesson 03의 렌더 루프에서 controls.update()와 같은 패턴입니다.

AnimationClip과 AnimationAction

Clip은 데이터, Action은 재생 제어입니다:

// gltf.animations = AnimationClip[]
// 각 clip을 action으로 변환
const clip = gltf.animations[0]  // "Idle" 등
const action = mixer.clipAction(clip)

// 재생
action.play()

// 정지
action.stop()

// 일시정지
action.paused = true

캔버스 오른쪽 상단의 버튼으로 각 애니메이션 클립을 전환해보세요.

크로스페이드 (부드러운 전환)

애니메이션을 갑자기 전환하면 부자연스럽습니다. fadeIn/fadeOut으로 부드러운 블렌딩을 합니다:

function fadeToAction(name, fadeDuration) {
  previousAction = activeAction
  activeAction = actions.get(name)

  // 이전 애니메이션 서서히 끄기
  previousAction.fadeOut(fadeDuration)

  // 새 애니메이션 서서히 켜기
  activeAction
    .reset()             // 처음부터
    .setEffectiveWeight(1) // 최대 무게
    .fadeIn(fadeDuration)  // 서서히 블렌딩
    .play()
}
  fadeOut(0.5):  ████████░░░░  (0.5초에 걸쳐 무게 1→0)
  fadeIn(0.5):   ░░░░████████  (0.5초에 걸쳐 무게 0→1)
                 ────────────
                 전환 구간 (0.5초)
                 두 애니메이션이 동시에 블렌딩됨

캔버스에서 다른 애니메이션 버튼을 클릭할 때 부드럽게 전환되는 것을 확인하세요.

Action 속성

action.timeScale = 2      // 2배속
action.timeScale = 0.5    // 0.5배속
action.timeScale = -1     // 역재생

action.weight = 1         // 최대 영향 (기본)
action.weight = 0.5       // 절반 블렌딩

action.loop = THREE.LoopRepeat    // 무한 반복 (기본)
action.loop = THREE.LoopOnce      // 1회만
action.loop = THREE.LoopPingPong  // 왕복

action.clampWhenFinished = true
// LoopOnce일 때 마지막 프레임에서 멈춤 (false면 초기 포즈로 돌아감)

캔버스 하단 버튼으로 속도와 루프 모드를 변경해보세요.

weight (무게) — 애니메이션 블렌딩

여러 애니메이션을 동시에 재생하면서 weight로 비율을 조절할 수 있습니다:

// 걷기 50% + 뛰기 50% = 빠르게 걷는 느낌
walkAction.weight = 0.5
runAction.weight = 0.5

// 걷기 100% + 손 흔들기 100% (상체만 영향)
// → 다른 부위에 영향하는 애니메이션이면 동시 재생 가능

스켈레탈 애니메이션의 원리

GLTF 모델의 캐릭터 애니메이션은 본(Bone) 기반입니다:

  SkinnedMesh (겉 표면)
       │
  Skeleton
  ├── spine (척추)
  │   ├── chest (가슴)
  │   │   ├── head (머리)
  │   │   ├── leftArm
  │   │   │   └── leftForeArm
  │   │   └── rightArm
  │   │       └── rightForeArm
  │   └── hips (엉덩이)
  │       ├── leftUpLeg
  │       │   └── leftLeg → leftFoot
  │       └── rightUpLeg
  │           └── rightLeg → rightFoot

  AnimationClip의 각 track이 특정 bone의
  position/rotation/scale을 시간에 따라 변경

Lesson 10의 씬 그래프와 같은 계층 구조입니다. 부모 bone이 회전하면 자식 bone도 따라 움직입니다.

AnimationClip의 내부

clip.name       // "Walk", "Run" 등
clip.duration   // 전체 길이 (초)
clip.tracks     // KeyframeTrack[] — 각 bone의 키프레임

// track 예시:
// "spine.quaternion" → 척추 bone의 회전
// "leftLeg.position" → 왼쪽 다리 bone의 위치
// 각 track에 시간별 keyframe 값이 저장됨

전체 코드

1단계: AnimationMixer 생성

loader.load('/models/Soldier.glb', (gltf) => {
  const model = gltf.scene
  scene.add(model)

  // mixer는 모델에 연결
  const mixer = new THREE.AnimationMixer(model)

  // 모든 clip을 action으로 등록
  const actions = new Map()
  gltf.animations.forEach((clip) => {
    actions.set(clip.name, mixer.clipAction(clip))
  })

  // 첫 애니메이션 재생
  actions.get('Idle')?.play()
})

2단계: 렌더 루프에서 업데이트

const clock = new THREE.Clock()

function animate() {
  requestAnimationFrame(animate)
  const delta = clock.getDelta()

  // mixer가 있으면 업데이트 (핵심!)
  if (mixer) mixer.update(delta)

  renderer.render(scene, camera)
}
animate()

getDelta()를 사용하는 이유: 프레임 간 실제 경과 시간을 전달해야 프레임 속도와 무관하게 일정한 애니메이션 속도가 보장됩니다.

이 레슨에서는 getElapsedTime()을 사용하지 않으므로 getDelta()와 혼용 문제가 없습니다.

3단계: 크로스페이드 전환

function fadeToAction(name: string, fadeDuration: number) {
  const action = actions.get(name)
  if (!action) return

  previousAction = activeAction
  activeAction = action

  // 이전 action fadeOut
  if (previousAction && previousAction !== activeAction) {
    previousAction.fadeOut(fadeDuration)
  }

  // 새 action fadeIn
  activeAction
    .reset()                          // 처음부터
    .setEffectiveTimeScale(timeScale) // 속도
    .setEffectiveWeight(1)            // 최대 무게
    .fadeIn(fadeDuration)             // 블렌딩
    .play()                           // 재생
}

.reset()은 action을 초기 상태로 되돌립니다. 이전에 재생이 끝난 action을 다시 시작할 때 필요합니다.

4단계: 속도 제어

// 속도 변경
action.setEffectiveTimeScale(2)    // 2배속
action.setEffectiveTimeScale(0.5)  // 0.5배속

// 일시정지/재개
action.paused = true   // 일시정지
action.paused = false  // 재개

5단계: 루프 모드

// 무한 반복 (기본)
action.loop = THREE.LoopRepeat

// 1회만 재생
action.loop = THREE.LoopOnce
action.clampWhenFinished = true
// clampWhenFinished: 마지막 포즈에서 멈춤

// 왕복 재생
action.loop = THREE.LoopPingPong

action.reset().play()  // 변경 후 재시작

6단계: 현재 상태 읽기

// 현재 재생 시간
action.time  // 초 단위

// 클립 정보
action.getClip().name      // 클립 이름
action.getClip().duration  // 전체 길이

// 현재 유효 무게 (fadeIn/Out 고려)
action.getEffectiveWeight()

// 현재 유효 속도
action.getEffectiveTimeScale()
