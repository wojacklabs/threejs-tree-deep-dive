01. Hello Three.js

Three.js란?

Three.js는 WebGL을 쉽게 사용할 수 있게 해주는 JavaScript 3D 라이브러리입니다. WebGL은 브라우저에서 GPU를 직접 사용해 3D 그래픽을 렌더링하는 저수준 API인데, 직접 사용하면 삼각형 하나 그리는 데도 수백 줄의 코드가 필요합니다.

Three.js는 이 복잡함을 추상화하여, 몇 줄의 코드로 3D 장면을 만들 수 있게 해줍니다.

최소 구성 3요소

Three.js로 무언가를 화면에 띄우려면 반드시 3가지가 필요합니다:

  Scene (장면)        Camera (카메라)      Renderer (렌더러)
  ┌──────────┐       ┌──────────┐        ┌──────────┐
  │ 오브젝트를  │       │ 어디서     │        │ 화면에     │
  │ 담는 컨테이너│       │ 바라볼지   │        │ 그리는 엔진 │
  └──────────┘       └──────────┘        └──────────┘
       │                   │                    │
       └───────────────────┴────────────────────┘
                           │
                    renderer.render(scene, camera)

Scene (장면)

THREE.Scene은 3D 세계의 컨테이너입니다. 모든 오브젝트, 조명, 카메라를 이 안에 추가(scene.add())해야 렌더링됩니다.

Camera (카메라)

PerspectiveCamera는 사람의 눈과 비슷한 원근 투영 카메라입니다.

4개의 파라미터를 받습니다:

  • fov (Field of View) — 시야각. 75도가 일반적
  • aspect — 종횡비 (가로/세로). 보통 캔버스 크기에 맞춤
  • near — 이것보다 가까운 건 안 보임 (0.1)
  • far — 이것보다 먼 건 안 보임 (1000)
     near        far
      │           │
      │  ╱────────╲
  카메라 ╱  보이는 영역  ╲
      │╲  (frustum)  ╱│
      │  ╲────────╱  │
      │           │
    0.1         1000
near/far와 성능: 이 범위 밖에 있는 오브젝트는 렌더링되지 않습니다. Three.js는 매 프레임마다 frustum culling을 수행하여, 카메라 시야(frustum) 밖의 오브젝트를 GPU에 보내지 않고 건너뜁니다. 따라서 far를 필요 이상으로 크게 잡으면 불필요한 렌더링이 발생합니다. 또한 near와 far의 비율이 너무 크면(예: 0.001 ~ 100000) z-fighting — 겹친 면이 깜빡이는 현상 — 이 발생할 수 있으므로, 필요한 범위만큼만 설정하는 것이 원칙입니다.

Renderer (렌더러)

WebGLRenderer는 Scene과 Camera의 정보를 바탕으로 실제로 픽셀을 그리는 엔진입니다.

HTML <canvas> 요소에 연결하여 그 위에 렌더링합니다.

Mesh = Geometry + Material

화면에 보이는 물체(Mesh)는 두 가지로 구성됩니다:

  • Geometry — 형태 (정육면체, 구, 원뿔 등의 꼭짓점 데이터)
  • Material — 외관 (색상, 질감, 반사 등)
  Geometry          Material           Mesh
  ┌─────┐          ┌─────┐          ┌─────┐
  │ 형태  │    +     │ 외관  │    =     │ 물체  │
  │ (꼭짓점)│          │ (색상) │          │       │
  └─────┘          └─────┘          └─────┘
BoxGeometry    MeshNormalMaterial     Mesh

MeshNormalMaterial

이 레슨에서는 MeshNormalMaterial을 사용합니다. 이 재질은 면의 법선(normal) 방향에 따라 색상이 자동으로 결정됩니다.

조명이 없어도 색이 보이므로 테스트에 적합하고, 회전하면 면마다 색이 바뀌어 3D 형태를 쉽게 확인할 수 있습니다.

왼쪽 캔버스에서 회전하는 정육면체를 확인하세요. 각 면의 법선 방향에 따라 빨강/초록/파랑 등의 색이 달라지는 것이 보입니다.

렌더 루프

requestAnimationFrame은 브라우저가 다음 화면을 그리기 직전에 함수를 호출해 달라고 예약하는 API입니다. 이걸 재귀적으로 호출하면 매 프레임마다 장면을 다시 그려서 애니메이션이 됩니다.

function animate() {
  requestAnimationFrame(animate)  // 다음 프레임 예약
  cube.rotation.y += 0.01         // 상태 변경
  renderer.render(scene, camera)  // 다시 그리기
}

전체 코드

이 레슨의 Three.js 코드 전문입니다. 각 단계를 주석으로 설명합니다.

1단계: import와 캔버스 연결

import * as THREE from 'three'

// HTML에 미리 만들어둔 <canvas id="c"> 를 가져옵니다
const canvas = document.getElementById('c') as HTMLCanvasElement

Three.js를 통째로 import합니다. THREE 네임스페이스 아래에 Scene, Camera, Geometry 등 모든 클래스가 있습니다.

2단계: Renderer 생성

const renderer = new THREE.WebGLRenderer({
  canvas,           // 어느 canvas에 그릴지
  antialias: true,  // 계단 현상 완화 (안티앨리어싱)
})

antialias: true를 주면 3D 물체의 가장자리가 부드러워집니다. 약간의 성능 비용이 있지만, 시각적 품질이 크게 향상됩니다.

3단계: Scene과 Camera

const scene = new THREE.Scene()

const camera = new THREE.PerspectiveCamera(
  75,   // fov: 시야각 75도
  1,    // aspect: 일단 1, resize에서 갱신
  0.1,  // near: 0.1 미만은 안 보임
  1000  // far: 1000 초과는 안 보임
)
camera.position.z = 3  // 카메라를 z축 +3 위치로 이동

camera.position.z = 3이 핵심입니다. 기본적으로 카메라와 오브젝트 모두 원점 (0,0,0)에 생성되므로, 카메라를 뒤로 빼야 오브젝트가 보입니다.

4단계: Mesh 생성 (Geometry + Material)

const geometry = new THREE.BoxGeometry(1, 1, 1)
// BoxGeometry(가로, 세로, 깊이) — 1x1x1 정육면체

const material = new THREE.MeshNormalMaterial()
// 법선 방향에 따라 색이 자동 결정 — 조명 불필요

const cube = new THREE.Mesh(geometry, material)
scene.add(cube)
// scene에 추가해야 렌더링됨!

5단계: 리사이즈 대응

function resize() {
  const parent = canvas.parentElement!
  const w = parent.clientWidth
  const h = parent.clientHeight
  renderer.setSize(w, h)
  renderer.setPixelRatio(Math.min(window.devicePixelRatio, 2))
  camera.aspect = w / h
  camera.updateProjectionMatrix()
}
window.addEventListener('resize', resize)
resize()  // 최초 1회 실행

브라우저 창 크기가 바뀔 때마다 renderer와 camera의 크기를 갱신합니다.

setPixelRatio는 고해상도 디스플레이(Retina)에서 선명하게 보이도록 합니다. 2 이상은 성능 낭비이므로 Math.min으로 제한합니다.

camera.updateProjectionMatrix()를 호출해야 변경된 aspect 비율이 실제로 적용됩니다.

6단계: 렌더 루프 (애니메이션)

function animate() {
  requestAnimationFrame(animate)

  cube.rotation.x += 0.01
  cube.rotation.y += 0.01

  renderer.render(scene, camera)
}
animate()

requestAnimationFrame은 보통 초당 60회 호출됩니다.

매 프레임마다 rotation을 조금씩 더하면 부드러운 회전 애니메이션이 됩니다.

0.01 라디안 ≈ 0.57도이므로, 360도 회전에 약 10.5초가 걸립니다.
