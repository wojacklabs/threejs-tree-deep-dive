포스트프로세싱이란?

3D 씬을 렌더링한 후 결과 이미지에 추가 효과를 적용하는 기법입니다. 카메라 필터처럼 최종 화면을 가공합니다.

  일반 렌더링:
  Scene → Camera → 화면

  포스트프로세싱:
  Scene → Camera → [텍스처] → Bloom → Film → ... → 화면
                   ↑ 여기서부터 2D 이미지 처리

EffectComposer

여러 포스트프로세싱 효과(Pass)를 체이닝하는 컨테이너입니다:

import { EffectComposer } from
  'three/examples/jsm/postprocessing/EffectComposer.js'

const composer = new EffectComposer(renderer)
composer.addPass(pass1)  // 순서대로 적용
composer.addPass(pass2)
composer.addPass(pass3)

// renderer.render() 대신:
composer.render()
포스트프로세싱 사용 시 renderer.render()를 composer.render()로 교체해야 합니다. 둘 다 호출하면 이중 렌더링이 됩니다.

Pass 파이프라인

각 Pass는 이전 Pass의 결과를 입력으로 받아 처리합니다:

  RenderPass (씬 렌더링)
      ↓ 텍스처
  UnrealBloomPass (발광)
      ↓ 텍스처
  FilmPass (필름 그레인)
      ↓ 텍스처
  OutputPass (톤매핑/색보정)
      ↓
  화면에 표시

RenderPass가 첫 번째, OutputPass가 마지막이어야 합니다.

UnrealBloomPass (발광)

가장 많이 사용되는 포스트프로세싱 효과입니다. 밝은 부분이 주변으로 번지는 발광 효과를 만듭니다.

new UnrealBloomPass(
  resolution,  // Vector2 — 렌더 해상도
  strength,    // 0~3 — 발광 강도
  radius,      // 0~1 — 번짐 반경
  threshold    // 0~1 — 이 밝기 이상만 bloom
)

캔버스 하단 슬라이더로 3가지 파라미터를 조절해보세요:

  • threshold — 낮추면 더 많은 부분이 발광. 0이면 전체가 빛남
  • strength — 발광의 강도. 높을수록 밝게 번짐
  • radius — 번짐의 범위. 높을수록 넓게 퍼짐
Bloom이 효과적이려면 발광할 오브젝트에 emissive와 높은 emissiveIntensity를 설정해야 합니다. threshold보다 밝은 부분만 bloom이 적용됩니다.

다른 내장 Pass들

캔버스의 Effects 버튼으로 켜고 끌 수 있습니다:

  Pass           효과                 용도
  ──────────────────────────────────────────────
  GlitchPass     화면 찢어짐/왜곡      공포, 사이버펑크
  FilmPass       필름 그레인/노이즈     레트로, 영화 느낌
  RGBShiftPass   색수차 (RGB 분리)     렌즈 효과
  DotScreenPass  도트 패턴             만화/팝아트
  BokehPass      피사계 심도 (DOF)     카메라 초점
  SSAOPass       간접 그림자            현실감 증가

ShaderPass — 커스텀 효과

내장 shader를 Pass로 감싸서 사용합니다:

import { ShaderPass } from
  'three/examples/jsm/postprocessing/ShaderPass.js'
import { RGBShiftShader } from
  'three/examples/jsm/shaders/RGBShiftShader.js'

const pass = new ShaderPass(RGBShiftShader)
pass.uniforms['amount'].value = 0.003
composer.addPass(pass)

자신만의 셰이더를 작성해서 ShaderPass에 넣으면 완전한 커스텀 포스트프로세싱도 가능합니다 (Lesson 16).

성능

포스트프로세싱은 전체 화면 픽셀을 처리하므로 비용이 있습니다:

  Pass 수        비용
  ──────────────────────────
  RenderPass만   기본 (추가 비용 없음)
  + Bloom        가벼움~보통 (다운샘플링)
  + SSAO         보통~무거움 (깊이 연산)
  + DOF          보통
  + 여러 개 조합   누적됨

  각 Pass마다 전체 화면 크기의 텍스처를 읽고 쓰므로
  Pass 수에 비례해서 비용 증가
모바일이나 저사양에서는 Bloom 하나만으로도 FPS가 떨어질 수 있습니다. EffectComposer 생성 시 해상도를 낮추거나( renderer.getPixelRatio()), Bloom의 resolution을 줄이는 것이 효과적입니다.

OutputPass — 왜 마지막에 필요한가?

포스트프로세싱 파이프라인은 리니어 색공간에서 작업합니다. OutputPass가 최종적으로 톤매핑과 sRGB 변환을 적용하여 화면에 올바른 색이 표시되도록 합니다.

// 마지막에 반드시 추가
composer.addPass(new OutputPass())

이것을 빠뜨리면 색이 너무 밝거나 씻겨 보입니다.

전체 코드

1단계: import

import { EffectComposer } from
  'three/examples/jsm/postprocessing/EffectComposer.js'
import { RenderPass } from
  'three/examples/jsm/postprocessing/RenderPass.js'
import { UnrealBloomPass } from
  'three/examples/jsm/postprocessing/UnrealBloomPass.js'
import { OutputPass } from
  'three/examples/jsm/postprocessing/OutputPass.js'
import { GlitchPass } from
  'three/examples/jsm/postprocessing/GlitchPass.js'
import { ShaderPass } from
  'three/examples/jsm/postprocessing/ShaderPass.js'
import { RGBShiftShader } from
  'three/examples/jsm/shaders/RGBShiftShader.js'

2단계: Composer 구성

const composer = new EffectComposer(renderer)

// 1. 씬 렌더링
composer.addPass(new RenderPass(scene, camera))

// 2. Bloom
const bloomPass = new UnrealBloomPass(
  new THREE.Vector2(width, height),
  0.8,   // strength
  0.4,   // radius
  0.85   // threshold
)
composer.addPass(bloomPass)

// 3~N. 추가 효과들
composer.addPass(new GlitchPass())

// 마지막. OutputPass
composer.addPass(new OutputPass())

3단계: 발광 오브젝트

// emissive + 높은 emissiveIntensity = bloom 대상
const neonMat = new THREE.MeshStandardMaterial({
  color: 0x111111,         // 기본 색 (어둡게)
  emissive: 0xff0088,      // 발광 색
  emissiveIntensity: 2,    // 발광 강도
})

// threshold보다 밝은 부분만 bloom
// emissiveIntensity가 높을수록 확실하게 bloom

일반 오브젝트(emissive 없음)는 bloom의 영향을 받지 않습니다. 캔버스에서 회색 큐브는 bloom이 없고, 네온 오브젝트만 빛납니다.

4단계: 렌더 루프 변경

function animate() {
  requestAnimationFrame(animate)

  controls.update()

  // renderer.render(scene, camera) ← 이전
  composer.render()  // ← 포스트프로세싱 사용 시
}
animate()

5단계: 리사이즈 처리

function resize() {
  const w = canvas.parentElement!.clientWidth
  const h = canvas.parentElement!.clientHeight
  renderer.setSize(w, h)
  camera.aspect = w / h
  camera.updateProjectionMatrix()

  // composer도 리사이즈!
  composer.setSize(w, h)
  // 또는 composer를 재생성
}

리사이즈 시 composer.setSize()를 호출하지 않으면 출력 해상도가 맞지 않아 화면이 흐려지거나 찌그러집니다.

6단계: Pass 런타임 제어

// Pass 활성화/비활성화
bloomPass.enabled = false  // bloom 끄기

// 파라미터 실시간 변경
bloomPass.strength = 1.5
bloomPass.radius = 0.6
bloomPass.threshold = 0.5

// ShaderPass uniform 변경
rgbShiftPass.uniforms['amount'].value = 0.005
