왜 GLTF인가?

3D 모델 포맷은 다양하지만(OBJ, FBX, Collada 등), glTF(GL Transmission Format)가 웹 3D의 표준입니다:

  포맷      장점                      단점
  ──────────────────────────────────────────
  glTF/GLB  웹 표준, 작은 크기,        ─
            PBR material, 애니메이션,
            Draco/Meshopt 압축 지원
  OBJ       단순, 널리 지원            material 제한, 애니메이션 없음
  FBX       Blender/Unity 호환         파일 큼, 파싱 복잡

.gltf는 JSON + 별도 바이너리/텍스처 파일, .glb는 모든 것이 하나로 합쳐진 바이너리 포맷입니다. 웹에서는 .glb가 추천됩니다 (파일 1개, 작은 크기).

GLTFLoader

Three.js addon으로 제공되는 GLTF 로더입니다:

import { GLTFLoader } from
  'three/examples/jsm/loaders/GLTFLoader.js'

const loader = new GLTFLoader()
loader.load(
  '/models/helmet.glb',   // 경로
  (gltf) => { ... },      // 성공 콜백
  (xhr) => { ... },       // 진행 콜백
  (error) => { ... }      // 에러 콜백
)

gltf 객체 구조

콜백으로 전달되는 gltf 객체에는 다양한 데이터가 들어있습니다:

gltf.scene       // Group — 모델의 루트 (scene.add 대상)
gltf.animations  // AnimationClip[] — 애니메이션 데이터
gltf.cameras     // Camera[] — 모델에 포함된 카메라
gltf.asset       // 메타데이터 (버전, 생성 도구 등)
gltf.scene은 THREE.Group입니다. 내부에 Mesh, Light, Bone 등이 트리 구조로 들어있습니다. Lesson 10에서 배운 씬 그래프가 여기서 실체화됩니다.

traverse()로 모델 수정

GLTF 모델의 모든 Mesh에 일괄 처리를 적용할 때 사용합니다:

model.traverse((child) => {
  if (child instanceof THREE.Mesh) {
    child.castShadow = true
    child.receiveShadow = true
    // material 수정, 교체 등
  }
})

캔버스의 Wireframe과 Normal View 버튼이 traverse로 모든 Mesh의 material을 일괄 변경하는 예시입니다.

모델 위치/스케일 조정

외부 모델은 원점, 스케일, 방향이 제각각이므로 조정이 필요합니다:

// 위치 조정
model.position.set(0, 0, 0)

// 스케일 조정 (너무 크거나 작을 때)
model.scale.setScalar(0.01)  // 균일 스케일
model.scale.set(0.01, 0.01, 0.01)  // 개별 축

// 회전 조정 (방향이 맞지 않을 때)
model.rotation.y = Math.PI  // 180도 회전
실무 팁: 모델이 로드 후 안 보이면 대부분 스케일 문제입니다. console.log(new THREE.Box3().setFromObject(model))로 바운딩 박스를 확인하고, 씬에 맞게 스케일을 조정하세요.

Draco 압축

Draco는 geometry 데이터를 60~90% 압축하는 Google 라이브러리입니다:

import { DRACOLoader } from
  'three/examples/jsm/loaders/DRACOLoader.js'

const dracoLoader = new DRACOLoader()
dracoLoader.setDecoderPath('/draco/')
// Draco WASM 디코더 경로

const gltfLoader = new GLTFLoader()
gltfLoader.setDRACOLoader(dracoLoader)
// GLTFLoader에 Draco 디코더 연결

Draco 압축된 GLB는 파일 크기가 크게 줄어들어 로딩 시간이 단축됩니다. 대규모 모델에서 특히 효과적입니다.

renderer.info

캔버스 왼쪽 상단에 표시되는 모델 정보(mesh 수, vertex 수, triangle 수)는 renderer.info에서도 확인할 수 있습니다:

console.log(renderer.info.render)
// { calls, triangles, points, lines, frame }

전체 코드

1단계: GLTFLoader import

import { GLTFLoader } from
  'three/examples/jsm/loaders/GLTFLoader.js'

const loader = new GLTFLoader()

Three.js 코어가 아닌 examples/jsm에서 import합니다. OrbitControls와 같은 경로 패턴입니다.

2단계: 모델 로드

loader.load(
  '/models/DamagedHelmet.glb',

  // onLoad — 로딩 완료
  (gltf) => {
    const model = gltf.scene
    model.position.set(0, 0.8, 0)
    scene.add(model)  // 씬에 추가
  },

  // onProgress — 로딩 중
  (xhr) => {
    if (xhr.lengthComputable) {
      const percent = (xhr.loaded / xhr.total) * 100
      progressBar.style.width = `${percent}%`
    }
  },

  // onError — 실패
  (error) => {
    console.error('Load error:', error)
  }
)

load()는 비동기입니다. 모델이 로드되기 전에 렌더 루프는 이미 돌고 있으므로, 씬에 추가되면 자동으로 표시됩니다.

3단계: 그림자 일괄 설정

model.traverse((child) => {
  if (child instanceof THREE.Mesh) {
    child.castShadow = true
    child.receiveShadow = true
  }
})

GLTF 모델은 기본적으로 그림자가 꺼져 있습니다. traverse로 모든 Mesh에 일괄 설정합니다.

4단계: 모델 정보 수집

let meshCount = 0
let vertexCount = 0
let triangleCount = 0

model.traverse((child) => {
  if (child instanceof THREE.Mesh) {
    meshCount++
    const geo = child.geometry
    vertexCount += geo.attributes.position.count

    if (geo.index) {
      triangleCount += geo.index.count / 3
    } else {
      triangleCount += geo.attributes.position.count / 3
    }
  }
})

indexed geometry는 geo.index.count / 3, non-indexed는 position.count / 3으로 삼각형 수를 계산합니다.

5단계: Material 교체 (디버깅)

// Wireframe 토글
model.traverse((child) => {
  if (child instanceof THREE.Mesh) {
    const mat = child.material as THREE.MeshStandardMaterial
    mat.wireframe = !mat.wireframe
  }
})

// NormalMaterial로 교체 (원본 저장)
model.traverse((child) => {
  if (child instanceof THREE.Mesh) {
    child.userData.originalMaterial = child.material
    child.material = new THREE.MeshNormalMaterial()
  }
})

// 원본 복원
model.traverse((child) => {
  if (child instanceof THREE.Mesh) {
    if (child.userData.originalMaterial) {
      child.material = child.userData.originalMaterial
    }
  }
})

userData는 임의 데이터를 저장할 수 있는 빈 객체입니다. 원본 material을 임시 저장하는 데 유용합니다.

6단계: Draco 압축 지원 (선택)

import { DRACOLoader } from
  'three/examples/jsm/loaders/DRACOLoader.js'

const dracoLoader = new DRACOLoader()
// CDN에서 디코더 로드
dracoLoader.setDecoderPath(
  'https://www.gstatic.com/draco/versioned/decoders/1.5.6/'
)

const gltfLoader = new GLTFLoader()
gltfLoader.setDRACOLoader(dracoLoader)
// 이후 동일하게 load() 호출

Draco 디코더는 WASM 기반이며, CDN에서 로드하면 별도 설치가 불필요합니다.
