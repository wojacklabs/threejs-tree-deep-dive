성능의 핵심: Draw Call

GPU 자체는 빠릅니다. 병목은 대부분 CPU→GPU 사이의 통신, 즉 draw call입니다.

  Draw Call = CPU가 GPU에게 "이거 그려" 명령 1회

  Mesh 2000개 (개별):
    CPU: "1번 그려" "2번 그려" ... "2000번 그려"
    = draw call 2000회 (CPU 병목!)

  InstancedMesh 1개:
    CPU: "이 geometry를 2000번 반복해서 그려"
    = draw call 1회

캔버스 하단 버튼으로 3가지 모드를 전환하면서 draw calls와 FPS를 비교해보세요.

3가지 최적화 전략 비교

  전략           draw calls  culling     개별 제어  색상    제한
  ────────────────────────────────────────────────────────
  Individual     N개         ✓ 개별      ✓ 완전   ✓ 개별  느림
  InstancedMesh  1개         ✗ 전체만    △ 행렬만  ✓ 개별  같은 geo
  Merged         1개         ✗ 전체만    ✗ 불가   ✗ 단색  같은 mat

InstancedMesh와 Merged는 draw call을 극적으로 줄이지만, frustum culling이 개별 단위로 안 됩니다. 인스턴스 1개만 화면에 보여도 전체가 GPU로 전송됩니다.

InstancedMesh

같은 geometry를 수천~수만 번 그릴 때 최적입니다. draw call 1회로 모든 인스턴스를 렌더링합니다:

const mesh = new THREE.InstancedMesh(
  geometry,  // 공유되는 형태
  material,  // 공유되는 재질
  count      // 인스턴스 수
)

// 각 인스턴스의 위치/회전/스케일을 행렬로 설정
const dummy = new THREE.Object3D()
dummy.position.set(x, y, z)
dummy.rotation.set(rx, ry, rz)
dummy.updateMatrix()
mesh.setMatrixAt(i, dummy.matrix)

// 인스턴스별 색상
mesh.setColorAt(i, new THREE.Color('red'))
InstancedMesh는 geometry와 material이 같아야 합니다. 다른 geometry를 섞으려면 여러 InstancedMesh가 필요합니다. 인스턴스별 색상은 setColorAt으로 가능하지만, 인스턴스별 다른 material은 불가능합니다.

InstancedMesh의 Culling 한계와 해결

InstancedMesh는 GPU 입장에서 오브젝트 1개입니다. 개별 인스턴스 중 1개만 화면에 걸쳐도 전체가 렌더링됩니다.

해결: 청크 분할

// 나쁨: 10만 개를 하나에 (culling 불가)
const mesh = new THREE.InstancedMesh(geo, mat, 100000)

// 좋음: 영역별로 나눠서 각각 culling
const chunkA = new THREE.InstancedMesh(geo, mat, 1000)
const chunkB = new THREE.InstancedMesh(geo, mat, 1000)
// 각 chunk의 bounding box가 작아서 culling 효과적

오픈월드 숲 같은 경우 영역별로 InstancedMesh를 나누면 화면 밖 청크가 통째로 culling됩니다. draw call은 청크 수만큼 늘지만, frustum 밖의 vertex를 GPU에 안 보내므로 전체적으로 더 빠릅니다.

Merged Geometry

여러 geometry를 하나로 합쳐 draw call을 줄입니다:

import { mergeGeometries } from
  'three/examples/jsm/utils/BufferGeometryUtils.js'

const geometries = []
for (let i = 0; i < 2000; i++) {
  const geo = baseGeo.clone()
  geo.translate(x, y, z)
  geo.rotateX(rx)
  geometries.push(geo)
}

const merged = mergeGeometries(geometries)
const mesh = new THREE.Mesh(merged, material)
// draw call 1회!
merge의 트레이드오프: draw call은 1이 되지만, 합친 후에는 개별 오브젝트를 이동/회전/삭제할 수 없습니다. 또한 같은 material만 가능하며, 거대한 하나의 mesh가 되므로 frustum culling이 안 됩니다 (Lesson 01에서 배운 것처럼 일부만 보여도 전체가 GPU로 전송).

LOD (Level of Detail)

카메라 거리에 따라 자동으로 디테일을 전환합니다. 캔버스 왼쪽의 빨간 구체에 줌 인/아웃해보세요:

const lod = new THREE.LOD()

// 가까울 때: 고폴리
lod.addLevel(highDetailMesh, 0)

// 15 이상: 중폴리
lod.addLevel(midDetailMesh, 15)

// 30 이상: 저폴리
lod.addLevel(lowDetailMesh, 30)

scene.add(lod)

info bar의 LOD level이 거리에 따라 변하는 것을 확인하세요.

renderer.info — 성능 측정

현재 렌더링 상태를 확인하는 핵심 도구입니다:

const info = renderer.info.render
info.calls      // draw call 수
info.triangles  // 렌더링된 삼각형 수
info.points     // 렌더링된 점 수
info.lines      // 렌더링된 선 수

renderer.info.memory
  .geometries   // GPU의 geometry 수
  .textures     // GPU의 텍스처 수

어디가 병목인지 파악

  증상                    원인              해결
  ──────────────────────────────────────────────
  draw calls 높음        오브젝트 수 많음   InstancedMesh, merge
  triangles 높음         vertex 많음       LOD, 저폴리, 디시메이션
  FPS 낮은데 calls 낮음  fragment 무거움    해상도↓, 셰이더 단순화
  메모리 부족            텍스처 큼          KTX2 압축, 해상도↓
CPU-bound vs GPU-bound: draw call을 줄여도 FPS가 안 오르면 GPU가 병목(GPU-bound)입니다. 이 경우 vertex/triangle 수를 줄이는 것이 훨씬 효과적입니다. 반대로 vertex가 적은데 느리면 draw call이 병목(CPU-bound)입니다. renderer.info의 calls와 triangles를 비교하여 어디가 병목인지 먼저 파악하세요.

기타 최적화 팁

  • renderer.setPixelRatio(Math.min(devicePixelRatio, 2)) — Retina에서 2 이상은 낭비
  • geometry.dispose(), material.dispose(), texture.dispose() — 불필요한 리소스 해제
  • 그림자는 필요한 오브젝트만 castShadow = true
  • 조명 수 최소화 (2~4개)
  • Object3D.frustumCulled = true (기본값) 유지
  • 텍스처 크기는 2의 거듭제곱 (밉맵 효율)

전체 코드

Individual — 개별 Mesh

for (let i = 0; i < 2000; i++) {
  const mat = new THREE.MeshStandardMaterial({
    color: new THREE.Color().setHSL(Math.random(), 0.7, 0.5),
  })
  const mesh = new THREE.Mesh(geo, mat)
  mesh.position.set(...)
  mesh.rotation.set(...)
  scene.add(mesh)
}
// draw calls: ~2000 (가장 느림)

각 mesh가 다른 material을 가지면 draw call이 더 늘어납니다. 같은 material을 공유해도 mesh 수만큼 draw call이 발생합니다.

InstancedMesh

const instancedMesh = new THREE.InstancedMesh(
  new THREE.BoxGeometry(0.5, 0.5, 0.5),
  new THREE.MeshStandardMaterial({ roughness: 0.5 }),
  2000  // 인스턴스 수
)

const dummy = new THREE.Object3D()
const color = new THREE.Color()

for (let i = 0; i < 2000; i++) {
  // 위치/회전을 행렬로 설정
  dummy.position.set(x, y, z)
  dummy.rotation.set(rx, ry, rz)
  dummy.updateMatrix()
  instancedMesh.setMatrixAt(i, dummy.matrix)

  // 인스턴스별 색상
  instancedMesh.setColorAt(i,
    color.setHSL(Math.random(), 0.7, 0.5)
  )
}

// GPU에 변경 알림
instancedMesh.instanceMatrix.needsUpdate = true
instancedMesh.instanceColor!.needsUpdate = true

scene.add(instancedMesh)
// draw calls: 1 (가장 빠름)

dummy.updateMatrix()를 호출해야 position/rotation이 matrix에 반영됩니다. setMatrixAt은 4×4 변환 행렬을 받습니다.

Merged Geometry

import { mergeGeometries } from
  'three/examples/jsm/utils/BufferGeometryUtils.js'

const geos: THREE.BufferGeometry[] = []
for (let i = 0; i < 2000; i++) {
  const clone = baseGeo.clone()
  clone.translate(x, y, z)  // geometry 자체를 이동
  clone.rotateX(rx)
  geos.push(clone)
}

const merged = mergeGeometries(geos)!
const mesh = new THREE.Mesh(merged, material)
scene.add(mesh)
// draw calls: 1, but no individual control

translate()와 rotateX()는 geometry의 vertex를 직접 변형합니다. mesh.position과 달리 되돌릴 수 없습니다.

LOD

const lod = new THREE.LOD()

const high = new THREE.Mesh(
  new THREE.SphereGeometry(2, 64, 32), mat
)
const mid = new THREE.Mesh(
  new THREE.SphereGeometry(2, 16, 8), mat
)
const low = new THREE.Mesh(
  new THREE.SphereGeometry(2, 6, 4),
  new THREE.MeshStandardMaterial({ flatShading: true })
)

lod.addLevel(high, 0)    // 거리 0~15: 고폴리
lod.addLevel(mid, 15)    // 거리 15~30: 중폴리
lod.addLevel(low, 30)    // 거리 30+: 저폴리

scene.add(lod)

// 현재 LOD 레벨 확인
lod.getCurrentLevel()  // 0, 1, 또는 2

LOD는 자동으로 카메라 거리를 계산하여 적절한 레벨을 표시합니다. 렌더 루프에서 별도 코드가 필요 없습니다.

FPS 측정

let fps = 0, frameCount = 0, lastTime = 0

function animate() {
  requestAnimationFrame(animate)

  frameCount++
  const now = performance.now()
  if (now - lastTime >= 500) {  // 0.5초마다 갱신
    fps = Math.round(
      frameCount / ((now - lastTime) / 1000)
    )
    frameCount = 0
    lastTime = now
  }
}

dispose() — 메모리 해제

// 모드 전환 시 이전 리소스 해제
function clearScene() {
  group.traverse((child) => {
    if (child instanceof THREE.Mesh) {
      child.geometry.dispose()
      child.material.dispose()
    }
  })
  scene.remove(group)
}
// dispose하지 않으면 GPU 메모리 누수!

씬에서 remove만 하면 GPU 메모리에 geometry/material이 남아있습니다. 반드시 dispose()를 호출해야 합니다.
