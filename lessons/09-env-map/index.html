09. 환경맵과 배경

다음 →

환경맵(Environment Map)이란?

환경맵은 오브젝트 주변 환경을 담은 텍스처입니다. 금속이나 유리처럼 반사/굴절하는 재질이 "무엇을 비추는지"를 결정합니다.

  환경맵 없이:              환경맵 있으면:
  ┌───────┐                ┌───────┐
  │       │                │ 주변   │
  │ 검은색 │                │ 풍경이  │
  │ 반사  │                │ 비침   │
  └───────┘                └───────┘
  금속인데 아무것도            실제 금속처럼 보임
  비치지 않음

캔버스 뒷줄에서 envMap 유무를 비교해보세요.

환경맵은 미리 만들어둔 고정 이미지입니다. 씬에 있는 다른 mesh(옆의 큐브, 캐릭터 등)는 반사에 보이지 않습니다. 오직 환경맵 이미지(하늘, 풍경 등)만 반사됩니다. 캔버스의 금속 구체에 옆의 빨간 기둥이나 초록 원뿔이 비치지 않는 이유가 이것입니다.

scene.background vs scene.environment

두 가지는 다른 역할을 합니다:

// 배경 이미지 (카메라 뒤에 보이는 것)
scene.background = envMap

// 모든 PBR material에 자동 적용되는 환경광
scene.environment = envMap
  scene.background     → 배경에 보이는 하늘/풍경
  scene.environment    → 오브젝트 반사에 사용되는 환경

  둘은 다른 텍스처를 쓸 수 있음:
  background = 흐린 풍경
  environment = 선명한 반사용 텍스처
scene.environment를 설정하면 모든 MeshStandardMaterial에 자동으로 envMap이 적용됩니다. 개별 material에서 envMapIntensity = 0으로 비활성화할 수 있습니다.

CubeTexture

환경맵의 가장 기본적인 형태입니다. 큐브의 6면 이미지로 구성됩니다:

const loader = new THREE.CubeTextureLoader()
const envMap = loader.load([
  'px.jpg', 'nx.jpg',  // +X, -X (좌우)
  'py.jpg', 'ny.jpg',  // +Y, -Y (상하)
  'pz.jpg', 'nz.jpg',  // +Z, -Z (앞뒤)
])
  큐브맵 6면:
         ┌──────┐
         │  +Y  │  (위: 하늘)
  ┌──────┼──────┼──────┬──────┐
  │  -X  │  +Z  │  +X  │  -Z  │
  └──────┼──────┼──────┴──────┘
         │  -Y  │  (아래: 땅)
         └──────┘

HDR 환경맵 (실무 표준)

실무에서는 CubeTexture 대신 HDR(High Dynamic Range) 이미지를 사용합니다:

import { RGBELoader } from
  'three/examples/jsm/loaders/RGBELoader.js'

const loader = new RGBELoader()
loader.load('/env.hdr', (texture) => {
  texture.mapping =
    THREE.EquirectangularReflectionMapping
  scene.background = texture
  scene.environment = texture
})

HDR의 장점:

  • 파일 1개 (6장이 아닌 equirectangular 파노라마)
  • 밝기 범위가 0~1을 넘어 현실적인 조명/반사
  • polyhaven.com에서 무료 HDR 제공

금속 재질

캔버스 앞줄에서 4가지 금속을 비교할 수 있습니다:

  재질       color      metalness  roughness
  ──────────────────────────────────────────
  크롬       #ffffff    1.0        0.0
  금         #ffd700    1.0        0.2
  알루미늄   #cccccc    0.9        0.5
  구리       #b87333    1.0        0.3

금속의 색상은 color로 결정됩니다. 크롬은 흰색(무채색 반사), 금은 노란 반사, 구리는 주황 반사.

MeshPhysicalMaterial — 유리

MeshPhysicalMaterial은 MeshStandardMaterial의 확장으로, 투과(transmission)와 굴절(refraction)을 지원합니다:

new THREE.MeshPhysicalMaterial({
  transmission: 1.0,  // 투과율 (0=불투명, 1=완전 투명)
  thickness: 1.5,     // 두께 (굴절 강도에 영향)
  ior: 1.5,           // 굴절률 (유리=1.5, 다이아=2.42)
  roughness: 0.0,     // 0=깨끗한 유리
})
MeshPhysicalMaterial은 MeshStandardMaterial보다 무겁습니다. transmission, clearcoat, sheen 등 고급 기능이 필요할 때만 사용하세요.

톤매핑 (Tone Mapping)

HDR 값(0~무한)을 화면에 표시할 수 있는 범위(0~1)로 변환하는 과정입니다. 캔버스의 ToneMap 버튼으로 5종을 비교할 수 있습니다:

  톤매핑       특징
  ──────────────────────────────────
  Linear       변환 없음 (HDR이 잘려서 하얗게 날아감)
  Reinhard     부드러운 롤오프
  Cineon       영화 필름 느낌
  ACESFilmic   영화 색감 (가장 많이 사용)
  AgX          최신, 색상 보존 우수

renderer.toneMapping = THREE.ACESFilmicToneMapping
renderer.toneMappingExposure = 1.0  // 노출 조절

주변 mesh를 진짜로 반사하려면?

고정 환경맵의 한계를 넘어 실시간 반사가 필요한 경우가 있습니다:

  방법              반사 대상        비용
  ──────────────────────────────────────────
  고정 envMap       환경 이미지만     거의 무료 (텍스처 읽기 1회)
  CubeCamera       주변 mesh 전체    6배 (6방향 렌더링)
  Reflector        주변 mesh 전체    2배 (1방향 렌더링)

CubeCamera는 오브젝트 위치에서 6방향으로 씬을 렌더링하여 실시간 큐브맵을 만듭니다. 크롬 구체처럼 모든 방향을 반사해야 할 때 사용합니다.

Reflector는 수면/거울처럼 평면 반사만 필요할 때 사용합니다. 카메라를 평면 기준으로 뒤집어 1번만 렌더링하므로 CubeCamera보다 훨씬 효율적입니다.

물 표면의 반사는 Reflector 또는 Three.js 내장 Water 클래스로 구현합니다. 물답게 보이려면 반사 외에도 프레넬 효과(정면은 투과, 측면은 반사), normal map 왜곡, vertex displacement(파도)가 필요합니다. 내장 Water는 데모 수준이며, 프로덕션에서는 커스텀 셰이더를 사용합니다 (Lesson 17).

전체 코드

1단계: 톤매핑 설정

const renderer = new THREE.WebGLRenderer({ canvas, antialias: true })
renderer.toneMapping = THREE.ACESFilmicToneMapping
renderer.toneMappingExposure = 1.0
// 톤매핑은 환경맵 사용 시 거의 필수
// exposure로 전체 밝기를 조절

2단계: 환경맵 생성/로딩

// 실제 프로젝트 — CubeTextureLoader
const cubeLoader = new THREE.CubeTextureLoader()
const envMap = cubeLoader.load([
  'px.jpg', 'nx.jpg',
  'py.jpg', 'ny.jpg',
  'pz.jpg', 'nz.jpg',
])

// 또는 HDR (추천)
import { RGBELoader } from
  'three/examples/jsm/loaders/RGBELoader.js'

new RGBELoader().load('env.hdr', (texture) => {
  texture.mapping =
    THREE.EquirectangularReflectionMapping
  scene.background = texture
  scene.environment = texture
})

이 레슨에서는 Canvas 2D로 프로시져럴 CubeTexture를 생성합니다. 실제 프로젝트에서는 위 코드를 사용하세요.

3단계: 배경과 환경 설정

// 배경 (카메라 뒤에 보이는 것)
scene.background = envMap

// 모든 PBR material에 자동 적용
scene.environment = envMap

// 개별 material에서 강도 조절
material.envMapIntensity = 1.5  // 반사 강하게
material.envMapIntensity = 0    // 반사 비활성화

4단계: 금속 재질

// 크롬 (완전 반사, 무채색)
new THREE.MeshStandardMaterial({
  color: 0xffffff,
  metalness: 1.0,
  roughness: 0.0,
})

// 금 (노란 반사)
new THREE.MeshStandardMaterial({
  color: 0xffd700,
  metalness: 1.0,
  roughness: 0.2,
})

// 구리 (주황 반사)
new THREE.MeshStandardMaterial({
  color: 0xb87333,
  metalness: 1.0,
  roughness: 0.3,
})

금속에서 color는 반사광의 색을 결정합니다. 비금속에서는 표면 색, 금속에서는 반사 색이 됩니다.

5단계: 유리 (MeshPhysicalMaterial)

new THREE.MeshPhysicalMaterial({
  color: 0xffffff,
  metalness: 0.0,
  roughness: 0.0,
  transmission: 1.0,  // 투과율 (유리의 핵심)
  thickness: 1.5,     // 내부 두께 (굴절 강도)
  ior: 1.5,           // 굴절률
  // 유리: 1.5
  // 물: 1.33
  // 다이아몬드: 2.42
})

// 착색 유리 — color로 색 지정
new THREE.MeshPhysicalMaterial({
  color: 0x4488ff,   // 파란 유리
  transmission: 0.9,
  ...
})

6단계: 톤매핑 변경 (런타임)

renderer.toneMapping = THREE.ACESFilmicToneMapping

// 변경 시 material 재컴파일 필요
scene.traverse((obj) => {
  if (obj instanceof THREE.Mesh) {
    obj.material.needsUpdate = true
  }
})
